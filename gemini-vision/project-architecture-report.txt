# UE AICartographer Plugin Project Architecture Report
Generated: 2026-04-12
Engine Version: Unreal Engine 5.3/5.4

## Overall Architecture
Hybrid Editor Plugin with Dual-Layer Design:
├─ [Layer 1] UE C++ Plugin (Plugins/AICartographer)
└─ [Layer 2] React Web Frontend (ue_mapping_plugin)

### Key Capabilities
- Editor-only plugin (WITH_EDITOR scope only)
- WebBrowser component integration for rendering React UI
- JSON-RPC bridge for bidirectional communication between UE and frontend
- AssetRegistry integration for project asset scanning
- Slate UI framework integration for native editor experience

## 2. Web Frontend Layer (ue_mapping_plugin)
Located at: D:\Amour\UEproject\Cropout\ue_mapping_plugin

### Core Technology Stack
- Build: Vite 8.x + TypeScript 5.x (strict mode)
- UI: React 18.x + TailwindCSS 4 + PostCSS

## 3. Cross-Layer Communication Flow
1. UE Editor ↔ C++ Bridge ↔ WebBrowser ↔ React Frontend
2. Protocol: JSON-RPC over postMessage interface
3. Data flow: Unidirectional state management (Zustand store single source of truth)

---

## 4. Critical Compliance Notes (per Project Rules)
### UE C++ Layer
- ✅ Editor-only API usage (no runtime pollution)
- ✅ TSharedPtr/Slate smart pointer usage (no raw pointers)
- ✅ Async AssetRegistry scanning (no blocking main thread)

### Frontend Layer
- ✅ Zustand centralized state management (no shared useState/Context)
- ✅ React Flow virtual DOM rendering (no native DOM manipulation)
- ✅ Async JSON parsing for large datasets (no render blocking)

---

## 5. Build & Deployment Pipeline
1. Frontend: `npm run build` → outputs to `dist/` folder
2. Manual copy: `dist/` contents → `Plugins/AICartographer/Resources/WebUI/`
3. UE: Compile plugin + load into editor
4. Access via editor menu: Window > Developer Tools > AICartographer
